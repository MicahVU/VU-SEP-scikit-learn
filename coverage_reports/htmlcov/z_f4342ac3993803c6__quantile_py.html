48class QuantileRegressor(LinearModel, RegressorMixin, BaseEstimator): 

49 """Linear regression model that predicts conditional quantiles. 

50 

51 The linear :class:`QuantileRegressor` optimizes the pinball loss for a 

52 desired `quantile` and is robust to outliers. 

53 

54 This model uses an L1 regularization like 

55 :class:`~sklearn.linear_model.Lasso`. 

56 

57 Read more in the :ref:`User Guide <quantile_regression>`. 

58 

59 .. versionadded:: 1.0 

60 

61 Parameters 

62 ---------- 

63 quantile : float, default=0.5 

64 The quantile that the model tries to predict. It must be strictly 

65 between 0 and 1. If 0.5 (default), the model predicts the 50% 

66 quantile, i.e. the median. 

67 

68 alpha : float, default=1.0 

69 Regularization constant that multiplies the L1 penalty term. 

70 

71 fit_intercept : bool, default=True 

72 Whether or not to fit the intercept. 

73 

74 solver : {'highs-ds', 'highs-ipm', 'highs', 'interior-point', \ 

75 'revised simplex'}, default='highs' 

76 Method used by :func:`scipy.optimize.linprog` to solve the linear 

77 programming formulation. 

78 

79 From `scipy>=1.6.0`, it is recommended to use the highs methods because 

80 they are the fastest ones. Solvers "highs-ds", "highs-ipm" and "highs" 

81 support sparse input data and, in fact, always convert to sparse csc. 

82 

83 From `scipy>=1.11.0`, "interior-point" is not available anymore. 

84 

85 .. versionchanged:: 1.4 

86 The default of `solver` changed to `"highs"` in version 1.4. 

87 

88 solver_options : dict, default=None 

89 Additional parameters passed to :func:`scipy.optimize.linprog` as 

90 options. If `None` and if `solver='interior-point'`, then 

91 `{"lstsq": True}` is passed to :func:`scipy.optimize.linprog` for the 

92 sake of stability. 

93 

94 Attributes 

95 ---------- 

96 coef_ : array of shape (n_features,) 

97 Estimated coefficients for the features. 

98 

99 intercept_ : float 

100 The intercept of the model, aka bias term. 

101 

102 n_features_in_ : int 

103 Number of features seen during :term:`fit`. 

104 

105 .. versionadded:: 0.24 

106 

107 feature_names_in_ : ndarray of shape (`n_features_in_`,) 

108 Names of features seen during :term:`fit`. Defined only when `X` 

109 has feature names that are all strings. 

110 

111 .. versionadded:: 1.0 

112 

113 n_iter_ : int 

114 The actual number of iterations performed by the solver. 

115 

116 See Also 

117 -------- 

118 Lasso : The Lasso is a linear model that estimates sparse coefficients 

119 with l1 regularization. 

120 HuberRegressor : Linear regression model that is robust to outliers. 

121 

122 Examples 

123 -------- 

124 >>> from sklearn.linear_model import QuantileRegressor 

125 >>> import numpy as np 

126 >>> n_samples, n_features = 10, 2 

127 >>> rng = np.random.RandomState(0) 

128 >>> y = rng.randn(n_samples) 

129 >>> X = rng.randn(n_samples, n_features) 

130 >>> # the two following lines are optional in practice 

131 >>> from sklearn.utils.fixes import sp_version, parse_version 

132 >>> solver = "highs" if sp_version >= parse_version("1.6.0") else "interior-point" 

133 >>> reg = QuantileRegressor(quantile=0.8, solver=solver).fit(X, y) 

134 >>> np.mean(y <= reg.predict(X)) 

135 0.8 

136 """
